# طريقة رفع تحديثات التطبيق

هذه الورقة تشرح كيف ترفع تحديثًا جديدًا للتطبيق بحيث يظهر للعميل داخل البرنامج.

## الفكرة

كل مرة تعدل البرنامج:

1. تغيّر رقم الإصدار
2. تحفظ التعديلات في Git
3. ترفعها إلى GitHub
4. تنشئ `tag` جديد للإصدار
5. GitHub يبني التحديث ويرفعه
6. العميل يرى رسالة: يوجد تحديث جديد

## قبل أن تبدأ

تأكد من هذه الأشياء:

- المشروع مربوط بـ GitHub
- GitHub Actions مفعّلة
- لديك إنترنت
- تعمل داخل مجلد المشروع الصحيح

## الخطوات في كل تحديث

### 1. عدّل البرنامج

أضف الميزة أو أصلح المشكلة التي طلبها العميل.

### 2. غيّر رقم الإصدار

افتح الملف:

[package.json](C:\Users\haied\Desktop\sj-project-v1.3\sj-project\package.json)

وابحث عن:

```json
"version": "1.0.0"
```

ثم غيّره إلى الإصدار الجديد.

مثال:

- من `1.0.0` إلى `1.0.1`
- أو من `1.0.1` إلى `1.0.2`

مهم:

- يجب تغيير رقم الإصدار كل مرة
- لا تكرر نفس الرقم

## أوامر التحديث

بعد إنهاء التعديل ورفع رقم الإصدار، افتح الطرفية داخل مجلد المشروع ونفّذ:

```bash
git add .
git commit -m "Update app to v1.0.1"
git push origin main
git tag v1.0.1
git push origin v1.0.1
```

## معنى هذه الأوامر

- `git add .`
  يجهز كل الملفات المعدلة

- `git commit -m "..." `
  يحفظ التعديل محليًا

- `git push origin main`
  يرفع التعديلات إلى GitHub

- `git tag v1.0.1`
  ينشئ إصدارًا جديدًا

- `git push origin v1.0.1`
  يرفع الإصدار إلى GitHub لكي يبدأ بناء التحديث

## مثال كامل

إذا عدلت البرنامج وأردت إصدارًا جديدًا برقم `1.0.2`:

1. غيّر `version` في `package.json` إلى `1.0.2`
2. ثم نفّذ:

```bash
git add .
git commit -m "Release v1.0.2"
git push origin main
git tag v1.0.2
git push origin v1.0.2
```

## أين أرى أن التحديث تم بنجاح

ادخل إلى GitHub ثم:

1. افتح المستودع
2. افتح `Actions`
3. انتظر حتى ينتهي البناء بنجاح
4. افتح `Releases`
5. ستجد الإصدار الجديد مثل `v1.0.2`

## متى يظهر التحديث للعميل

يظهر التحديث إذا:

- العميل لديه إصدار أقدم
- جهازه متصل بالإنترنت
- فتح التطبيق

عندها سيظهر له تنبيه بأن هناك تحديثًا جديدًا.

## إذا فشل التحديث

راجع هذه الأشياء:

- هل غيّرت `version` في `package.json`؟
- هل أنشأت `tag` بنفس الرقم؟
- هل رفعت `tag` إلى GitHub؟
- هل نجح البناء في `Actions`؟
- هل ظهر الإصدار في `Releases`؟

## قاعدة سريعة جدًا

كل تحديث = 4 خطوات:

1. عدّل البرنامج
2. غيّر رقم النسخة
3. `commit` و `push`
4. `tag` و `push`

## أرقام الإصدارات

أمثلة:

- `1.0.0`
- `1.0.1`
- `1.0.2`
- `1.1.0`
- `2.0.0`

اقتراح بسيط:

- إصلاح صغير: زد الرقم الأخير
- إضافة متوسطة: زد الرقم الأوسط
- تغيير كبير: زد الرقم الأول
